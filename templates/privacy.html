Our Commitment

PhishGuard is an AI-powered phishing email detection tool. We take your privacy seriously. This policy explains what data we access, how we use it, and what we never do with it.

// key principle
PhishGuard does not store, log, or share the content of any email you analyze. Email text is processed in real time and immediately discarded after analysis.
// data_access

What We Access

When you use the PhishGuard Chrome extension on Gmail, we access the following:

  • The subject line of the email you are currently viewing
  • The sender's name and email address
  • The body text of the email you are currently viewing

This content is accessed only when you open an email in Gmail and only for the purpose of phishing analysis.

// data_usage

How We Use Your Data

The email content accessed by the extension is sent to the PhishGuard API for analysis. The analysis involves:

  • Rule-based checks for suspicious patterns (urgency language, spoofed domains, shortened URLs)
  • A machine learning classifier trained on publicly available phishing datasets
  • An AI language model (Llama 3.3-70B via Groq API) for contextual reasoning

The analysis result — a risk score, verdict, and explanation — is returned to your browser and displayed in the extension. The email content is not retained after analysis is complete.

// data_we_never_collect

What We Never Do

  • We never store or log the content of your emails
  • We never store your Gmail credentials or authentication tokens
  • We never share your email content with third parties for advertising
  • We never build profiles or track your behavior across sessions
  • We never sell your data to any third party
  • We never access emails you have not opened
// third_party_services

Third-Party Services

PhishGuard uses the following third-party services to perform analysis:

  • Groq API — used to run the Llama 3.3-70B language model for AI reasoning. Email text is sent to Groq's API for inference. Groq's privacy policy applies: groq.com/privacy-policy
  • Render — our API is hosted on Render's cloud infrastructure. No email data is persisted on Render's servers.

We recommend reviewing Groq's privacy policy to understand how inference requests are handled on their end.

// data_retention

Data Retention

PhishGuard retains no email data. Each analysis request is stateless — the email text is sent, analyzed, and the result is returned. Nothing is written to a database or log file.

The most recent analysis result is temporarily stored in your browser's local extension storage (chrome.storage.local) so the popup can display it. This data lives only in your browser and is cleared when you close Chrome or install a new version of the extension.

// extension_permissions

Why We Need These Permissions

  • Access to mail.google.com — required to read email content for analysis
  • Storage — stores the last analysis result locally in your browser for the popup display
  • Active tab — required to inject the analysis banner into the Gmail page
  • Tabs — required to open the full PhishGuard web app in a new tab
